Elements API

This documentation is designed for those who want to add new elements.

You need some tools in some steps. You'll find them here:

  • Create new package: Tools for contributors\Element creation\Create new package.ahk
  • Element template creator: Tools for contributors\Element creation\Element template creator.ahk
  • Lang crawler: language\lang crawler.ahk
  • Translation tool: language\Translation Tool.ahk
  • Find modules: find modules.ahk
  • InnoSetup package creator pre build steps: innoSetupPackageCreatorPreBuildSteps.ahk

How to start

AHF is designed to make it as easy as possible to create new elements or even extension packages.

You'll find here multiple instructions depending on what you want to do.

  • If you want to create an element just for you, read Create new elements with low effort.
  • If you want to add elements that are not very special and don't require external applications, consider to submit them to the default package. Read Create new elements for the default package.
  • If you want to add elements for an external application or with some very special stuff, consider to create a new package with an installer. Read Create a new package with installer.

Create new elements with low effort

This section describes how you can create new elements with minimum effort.

  • Create a new package using the tool Create new package
    • Enter a package name of your choice
  • Create a new element using the tool Element template creator
    • Choose the package
    • Choose the element type
    • Enter a name
    • Choose or enter a category name
    • Choose which kind of parameters you want to have
    • Choose which kind of pre coded features you want to have
    • Click on Generate file
  • If your element requires a new library
    • Add the lib file to source_elements\Default\lib\%anyNewSubFolder%
  • Open the file manifest.json in the folder source_elements\%yourPackageName%
    • Add the name of the new element in the list of the matching element type
    • Add the file path of the new library (if any) in section libraries
  • Run the tool Find modules. This will make the source code files available to AHF
  • Open the source file of the newly created element. You'll find it in the folder source_elements\%yourPackageName%\%ElementType%s
    • Edit the code and implement everything. See other help pages for help
  • Restart AHF. The new element will be available now

Create new elements for the default package

This section describes how you can create new elements that may be submitted to the default package of AHF

  • Create a new element using the tool Element template creator
    • Choose the element type
    • Enter a name
    • Choose or enter a category name
    • Choose an icon. You may also create a new icon in Source_Elements\Default\Icons. It will be then selectable here.
    • Choose the stability level.
    • Choose which kind of parameters you want to have
    • Choose which kind of pre coded features you want to have
    • Click on Generate file
  • If your element requires a new library
    • Add the lib file to source_elements\Default\lib\%anyNewSubFolder%
  • Open the file manifest.json in the folder source_elements\Default
    • Add the name of the new element in the list of the matching element type
    • Add the file path of the new library (if any) in section libraries
  • Run the tool Find modules. This will make the source code files available to AHF
  • Open the source file of the newly created element. You'll find it in the folder source_elements\Default\%ElementType%s
    • Edit the code and implement everything. See other help pages for help
  • Restart AHF. The new element will be available now
  • Test your element. It's best if you create a flow that tests it with different configurations
  • Write a help file for your element.
    • Go to the folder Help and copy the file template.html
    • Paste it to source_elements\Default\help\en\%ElementType%s and paste it there.
    • Rename it to the element name.
    • Open and edit it. Describe it alike other elements in the default package.
  • Run the tool Find modules. This will update make the help file available to AHF
  • Restart AHF. Test whether the help file works from the element editor window
  • Submit the element via forum, via email to autohotflow@arcor.de or via merge request on git. It should include
    • The source file of the element
    • The help file
    • The test flow
    • Library file, (if any)

Create a new package with installer

This section describes how you can create a new package with an installer.

The installer generator is not finished yet. It will follow soon.

You need to install the InnoSetup application. It can be downloaded here: https://jrsoftware.org/isdl.php

  • Create a new element using the tool Element template creator
    • Choose the element type
    • Enter a name
    • Choose or enter a category name
    • Choose an icon. You may also create a new icon in Source_Elements\%yourPackageName%\Icons. It will be then selectable here.
    • Choose the stability level.
    • Choose which kind of parameters you want to have
    • Choose which kind of pre coded features you want to have
    • Click on Generate file
  • If your element requires a new library
    • Add the lib file to source_elements\Default\lib\%anyNewSubFolder%
  • Open the file manifest.json in the folder source_elements\%yourPackageName%
    • Add the name of the new element in the list of the matching element type
    • Add the file path of the new library (if any) in section libraries
  • Run the tool Find modules. This will update the source code files of AHF
  • Open the source file of the newly created element. You'll find it in the folder source_elements\%yourPackageName%\%ElementType%s
    • Edit the code and implement everything. See other help pages for help
  • Restart AHF. The new element will be available now
  • Test your element. It's best if you create a flow that tests it with different configurations
  • Run Lang crawler. This will find all translatable strings in your package and write them in source_elements\%yourPackageName%\language\en.ini
  • You may use Translation tool and translate everything to an other language
  • Build the setup file
    • Use tool InnoSetup package creator pre build steps to update the InnoSetup file innoSetupPackageInstallerScript.iss. You'll have to select your package name.
    • Open innoSetupPackageInstallerScript.iss and compile the installer
  • Publish the package installer on your own via forum and notify bichlepa via email to autohotflow@arcor.de
