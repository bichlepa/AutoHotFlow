Parametrization details

Definition of parametrization details and defining the content of element settings window.

The function Element_getParametrizationDetails_...(Environment) returns all informations about the parameters and how they can be changed by the user in the element settings window.
It must return an array where each array entry defines a line in the element settings window.
It's recommended to use the x_lang() function for all labels to allow the text to be translated in other languages.

Type: Label

This entry shows a label in the element settings window.

label

Text of the label.

size optional

Defines the size of the label.

    Possible values:

  • empty (default size)

  • small

id optional

Defines the ID of the label. This ID allows to change the label text in function Element_CheckSettings_...

color optional

Defines the color of the label.

Type: checkbox

This entry shows a checkbox in the element settings window.

The parameter value can be 0 (checked), 1 (unchecked) or -1 (gray state, neither checked nor unchecked)).

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

default

Default value of the parameter.

label

Text of the label.

gray optional

If true the checkbox can be set to gray state (neither selected nor unselected, returns value -1).

Type: radio

This entry shows a radio group in the element settings window.

The parameter value can be either the index or the enum value of the selected entry.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

default

Default value of the parameter. It is affected by the option result.

choices

Array with label text of all choices.

result optional

Defines what kind of value the parameter gets.

    Possible values:

  • empty (The parameter value is the index of the selected item)

  • enum (The parameter value is the enumeration ID of the selected item)

enum required if result = enum

Array with the enumeration IDs of all choices. The order must be same as in choices.

Type: edit

This entry shows edit control in the element settings window.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...
It can be an array with multiple IDs. This will show multiple edit fields side by side.

default

Default value of the parameter.

content optional

Content type of the parameter.
A correspoding icon and help text will be shown near the edit field.
The API functions which automatically evaluate parameters use this option to find out how to evaluate it.
This option can contain an array of those values. If so, it will show a radio group where the user can select the desired content type.

    Possible values:

  • String (Normal string. Content of parameters can be inserted using the percent signs)

  • RawString (Raw string. Content of parameters can't be passed here)

  • VariableName (Name of a variable)

  • Expression (Normal expression with any result value type)

  • Number (Expression which must result to a number)

  • Integer (Expression which must result to an integer)

  • PositiveInteger (Expression which must result to an integer. 1 or greater)

  • positiveIntegerOrZero (Expression which must result to an integer. 0 or greater)

ContentID required if content is an array

ID of the parameter which stores the selected content ID.

ContentDefault required if content is an array

Default content type.

WarnIfEmpty optional

Whether value must not be empty.
A correspoding icon and help text will be shown near the edit field if it is empty.
The API functions which automatically evaluate parameters uses this option.

    Possible values:

  • false or empty (Value can be empty)

  • true (Value must not be empty)

useupdown optional

If true it shows up and down buttons near the edit field where user can change the value by 1.

range optional or useupdown = true

Allowed range of values which can be set via the up and down buttons.

Type: slider

This entry shows a slider where user can set a number. It can optionally show an edit field nearby.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

default

Default value of the parameter.

allowExpression

If true an edit control will be shown near the slider to allow the user to enter an expression.

content optional if allowExpression = true

Content type of the parameter.
A correspoding icon and help text will be shown near the edit field.
The API functions which automatically evaluate parameters use this option to find out how to evaluate it.
This option can contain an array of those values. If so, it will show a radio group where the user can select the desired content type.

  • Integer (Expression which must result to an integer)

  • PositiveInteger (Expression which must result to an integer. 1 or greater)

  • positiveIntegerOrZero (Expression which must result to an integer. 0 or greater)

options

Additional slider options as defined in AutoHotkey language.

    example:

  • Range0-255 tooltip tickinterval10

This entry shows a button in the element settings window.

label

Text of the button.

Defines the ID of the button. This ID allows to change the label text in function Element_CheckSettings_...

Defines the function name or goto-label that will be called if user clicks on the button.
It's recommended to use a function. If the function may have one parameter, the parameter will contain the "Environment" value that may be used in some API functions.

Type: file

This entry shows an edit field, where user can enter a file path, and a button, which opens the file select dialog, in the element settings window.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

label

Prompt that will be shown in the file select dialog.

filter

Filter of the file select dialog.

    example:

  • x_lang("Images and icons") " (*.gif; *.jpg; *.bmp; *.ico; *.cur; *.ani; *.png; *.tif; *.exif; *.wmf; *.emf; *.exe; *.dll; *.cpl; *.scr)"

options

Additional options for the AutoHotkey command FileSelectFile as defined in AutoHotkey language.

    example:

  • 8

Type: weekdays

This entry shows some checkboxes, where user can select weekdays, in the element settings window.
The parameter value is a string with all selected weekdays where each weekday is represented by a number. For example: 147 means monday, thursday and sunday are selected.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

default

Default value of the parameter.

WarnIfEmpty optional

Whether value must not be empty.
A correspoding icon and help text will be shown near the edit field if it is empty. (not yet implemented)
The API functions which automatically evaluate parameters uses this option.

    Possible values:

  • false or empty (Value can be empty)

  • true (Value must not be empty)

Type: dateAndTime

This entry shows a DateTime control, where user can select a time.
The parameter value is a timestamp in the YYYYMMDDHH24MISS format.

id

The ID of the parameter. This ID allows to perform actions via API calls in Element_CheckSettings_...

default

Default value of the parameter.

format optional

Format of the control.

    Possible values:

  • Time (Control shows only the time without the date)

  • Date (Control shows only the date)

  • DateTime (Control shows the time and the date)
