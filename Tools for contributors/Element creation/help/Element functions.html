Element functions

The template has some functions which must be present in every element implementation.
The three dots ... are placehoders for the unique element class which usually consists of element type and sub type. If you build a package, it is recommended to add a prefix to the sub type.
Some function have the parameter Environment. This variable must never be changed and is not meant to be read. It must be passed to some API functions.

Element_getName_...()

This function returns the name of the element which is shown to the user in the element subtype selection menu.
It's recommended to use the x_lang() function to allow the text to be translated in other languages.

Element_getCategory_...()

This function returns the categories of the element. The categories define the folder where the element will be put in in the subtype selection menu.
Usually each element has a single category. They can also have multiple categories, it will then show up in multiple folders. Use a pipe | to separate the categories.
It's recommended to use the x_lang() function to allow the text to be translated in other languages.

Element_getElementLevel_...()

This function returns required experience level of the user. It can be one of following values: Beginner, Advanced or Programmer

Element_getIconPath_...()

This function returns the path of the icon. The origin path is the "icons" folder inside the package folder.
If you want to reuse an icon from the default package, you have to copy the icon file into your package.

Element_getStabilityLevel_...()

This function returns the stability level of the element. It can be one of following values: Stable or Experimental
Stable means that the element doesn't have known bugs and is fully implemented. It's also granted that there won't be any compatibility breaking changes in the future.
Experimental means that the element has either known bugs or is not fully implemented. Future updates may also break the compatibility.

Element_getParametrizationDetails_...(Environment)

This function returns the parameters of the element with details how to show them in the element settings window.
They are explained below.

Element_GenerateName_...(Environment, ElementParameters)

This function returns the individual element name which can vary depending on the parameter values the user has set.
It can use some element API functions. They are described below.
It's recommended to use the x_lang() function to allow the text to be translated in other languages.

ElementParameters contains the element parameter values.

Element_CheckSettings_...(Environment, ElementParameters, staticValues)

This function can check the parameters while user edits them in the element settings window.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.
staticValues is an object which will keep data that is stored inside it while the element settings window is open.

Element_run_...(Environment, ElementParameters) only actions, conditions and loops

This function executes the element.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.

Element_stop_...(Environment, ElementParameters) only actions, conditions and loops

This function stops the execution of the element.
If a function can take a long time, it it good practice, if it is possible to stop the execution via this function.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.

Element_enable_...(Environment, ElementParameters) only triggers

This function enables a trigger. It must return true if enabling was successful.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.

Element_postTrigger_...(Environment, ElementParameters, TriggerData) only triggers

This function is called after a trigger event occured and a new execution instance has been created.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.

TriggerData may contain data which were passed with the trigger event.

Element_disable_...(Environment, ElementParameters) only triggers

This function disables a trigger.
It can use some element API functions. They are described below.

ElementParameters contains the element parameter values.

Remarks

From AHF Version 1.2, following functions became obsolete and can be removed: x_RegisterElementClass("..."), Element_getElementType_...(), Element_getPackage_...()
