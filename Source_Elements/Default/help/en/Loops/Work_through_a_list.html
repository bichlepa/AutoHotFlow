Work through a list

The loop Work through a list allows to loop over all elements in a list, one at a time.

Settings

Variable name

Name of the variable that contains a list.

Performance

If Copy list before first iteration is checked, the list will be copied prior to first iteration. This might slightly increase the performance and be more convenient.
If not, the elements will be retrieved directly from list on each loop.

Note!

This might cause trouble if you delete or add an element inside the loop. For example:

Example:

A list contains following elements: ONE, TWO, THREE

On first iteration a_LoopValue contains ONE. Inside the loop, an action is used, that deletes the first element. On second iteration a_LoopValue will contain THREE. After that iteration the loop will end.


Supplied variables

A_Index

contains the number of the current iteration. It starts with the value 1 and increases on every loop.

A_LoopKey

contains the current key.
In most lists, that only contain numerical keys, this value will be same as a_index.

A_LoopValue

contains the current value. That is one element of the list.

A_LoopUseCopiedList

contains 0 ore 1, depending on whether Copy list before first iteration is checked.

A_LoopCurrentList

if Copy list before first iteration is checked, this variable will contain the copied list.
