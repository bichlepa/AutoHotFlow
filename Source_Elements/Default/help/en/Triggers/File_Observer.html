File observer

The trigger File observer monitors a folder for changes. It triggers the flow ss soon as a file or folder is modified.

Settings

Path to observer

Specify the directory which will be observed.

Subfolders

If Include subfolders is selected, all changes in the subfolders will be detected, too.

Event types

Specify which changes should trigger the flow. There are following options:

  • File created: Any change to the creation time of files in the watched directory or subtree will trigger the flow.

  • File accessed: Any change to the last access time of files in the watched directory or subtree will trigger the flow.

  • Written to file: Any change to the last write-time of files in the watched directory or subtree will trigger the flow. The operating system detects a change to the last write-time only when the file is written to the disk. For operating systems that use extensive caching, detection occurs only when the cache is sufficiently flushed.

  • File size changed: Any file-size change in the watched directory or subtree will trigger the flow. The operating system detects a change in file size only when the file is written to the disk. For operating systems that use extensive caching, detection occurs only when the cache is sufficiently flushed.

  • File name changed: Any file name change in the watched directory or subtree will trigger the flow. Changes include renaming, creating, or deleting a file.

  • Folder name changed: Any directory-name change in the watched directory or subtree will trigger the flow. Changes include creating or deleting a directory.

  • Attributes changed: Any attribute change in the watched directory or subtree will trigger the flow.

  • Security changed: Any security-descriptor change in the watched directory or subtree will trigger the flow.

Supplied variables

A_FileChangeType

contains an event type. Unlike the event types which can be selected above, this variable will contain one of following strings:

  • ADDED: A new file was created.

  • REMOVED: A file was removed.

  • RENAMED: A file was renamed.

  • MODIFIED: A file's content or attributes were changed.

A_FileChangedFrom and A_FileChangedTO

contain the file names.

If a file was added, A_FileChangedFrom will be empty and A_FileChangedTO will contain the file name.
If a file was removed, A_FileChangedFrom will contain the file name and A_FileChangedTO will be empty.
If a file was renamed, A_FileChangedFrom will contain the old name and A_FileChangedTO will contain the new name.
If a file was modified, A_FileChangedFrom and A_FileChangedTO will contain the file name.
