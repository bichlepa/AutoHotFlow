Send keystrokes to a control

The action Send keystrokes sends simulated keystrokes and mouse clicks to the active window.
If no window is specified, the currently active window will be found.

Settings

Keys or text to send

Specify the keys that will be sent.
If Raw mode is selected, the entered string will be sent as is.
If Raw mode is not selected, some keys can be declared as described in the section Remarks.

Output variable name

Name of the variable which will contain the retrieved control text.

Method for control identification

The control can be identified either by its text or by its classname and instance number.

Text in control or Classname and instance number of the control

Enter here either the text or the classname and instance number of the control, as specified above.

Title of window

Name of the window title. It can be chosen whether the title starts, contains or ends with the specified string.

Exclude title

Windows whose titles include this value will not be considered.

Text of a control in window

If present, this parameter must be a substring from a single text element of the target window.
It can be specified whether hidden text will be found.

Exclude text of a control in window

Windows whose text include this value will not be considered.

Window class

A window class is a set of attributes that the system uses as a template to create a window. In other words, the class name of the window identifies what type of window it is.

Process name

The name or path of the executable that created that window.

Unique window ID

Each window or control has a unique ID, also known as a HWND (short for handle to window). This ID can be used to identify the window or control even if its title changes.

Unique process ID

Use the unique process id to identify a window belonging to a specific process. Each process has its unique ID, even if it has the same executable.

Hidden window

If detect hidden window is selected, a window that exists but is not visible will be found.

Wizard to get parameters

Use the wizard to get parameters for your window. Follow the instruction in the tooltip near to the mouse.

Remarks

When not in raw mode, the following characters are treated as modifiers (these modifiers affect only the very next key):

!

Sends an ALT keystroke. For example, This is text!a would send the keys "This is text" and then press ALT+a. Note: !A produces a different effect in some programs than !a. This is because !A presses ALT+SHIFT+A and !a presses ALT+a. If in doubt, use lowercase.

+

Sends a SHIFT keystroke. For example, +abC would send the text "AbC", and !+a would press ALT+SHIFT+a.

^

Sends a CONTROL keystroke. For example, ^!a would press CTRL+ALT+a, and ^{Home} would send CONTROL+HOME. Note: ^A produces a different effect in some programs than ^a. This is because ^A presses CONTROL+SHIFT+A and ^a presses CONTROL+a. If in doubt, use lowercase.

#

Sends a WIN keystroke, therefore #e would hold down the Windows key and then press the letter "e".

{ASC nnnnn}

Sends an ALT+nnnnn keypad combination, which can be used to generate special characters that don't exist on the keyboard. To generate ASCII characters, specify a number between 1 and 255. To generate ANSI characters (standard in most languages), specify a number between 128 and 255, but precede it with a leading zero, e.g. {Asc 0133}.

Unicode characters may be generated by specifying a number between 256 and 65535 (without a leading zero). However, this is not supported by all applications. For alternatives, see the section below.

{U+nnnn}

Sends a Unicode character where nnnn is the hexadecimal value of the character excluding the 0x prefix. This typically isn't needed in Unicode versions of AutoHotkey, where Send and ControlSend automatically support Unicode text.

If the character doesn't map to a virtual keycode, it will be sent via input mode and the specified Send mode has no effect.

{vkXX}
{scYYY}
{vkXXscYYY}

Sends a keystroke that has virtual key XX and scan code YYY. For example: Send {vkFFsc159}. If the sc or vk portion is omitted, the most appropriate value is sent in its place.

The values for XX and YYY are hexadecimal and can usually be determined using AutoHotKey from the main window's View→Key history menu item.

{Blind}

When {Blind} is the first item in the string, the program avoids releasing Alt/Control/Shift/Win if they started out in the down position.

{Blind} also causes the state of Capslock not to be changed. Finally, {Blind} omits the extra Control keystrokes that would otherwise be sent; such keystrokes prevent: 1) Start Menu appearance during LWin/RWin keystrokes; 2) menu bar activation during Alt keystrokes.

{Blind} is not completely supported by SendPlay, especially when dealing with the modifier keys (Control, Alt, Shift, and Win).

{Raw}

Sends the keystrokes exactly as they appear rather than translating {Enter} to an ENTER keystroke, ^c to Control-C, etc. Although the string {Raw} need not occur at the beginning of the string, once specified, it stays in effect for the remainder of the string.
