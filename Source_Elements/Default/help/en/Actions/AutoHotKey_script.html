AutoHotKey script

The action AutoHotKey script creates a new AutoHotKey scripts and executes it. Don't use that action if you don't master the AutoHotKey scripting language.
Note!

This action is experimental.

This action has low performance, since a new instance of AutoHotKey must be loaded.

Settings

AutoHotKey script

Script that will be executed.

Variables that should be exported prior to execution

Specify a list of variables that should be exported to the AutoHotKey script prior to execution. Those variables will be accessible in the script.
The list can be separated either by a space, linefeed, comma or tab.

Variables that should be imported after execution

Specify a list of variables that should be imported from the AutoHotKey script after execution. Those variables will be accessible in the flow.

The list can be separated either by a space, linefeed, comma or tab.

Remarks

If the script should end with the result exception then the variable result must have the value exception. (result=exception)

If the script reaches the bottom, it will end. If the script is persistent (e.g. it uses timers, GUIs, hotkeys …) then the script should be exited by jumping to the label endExecution. (goto endExecution). Otherwise, the flow will not notice that the script has finished.
