AutoHotKey script

The action AutoHotKey script executes an AutoHotkey script in a new AutoHotkey thread.

Don't use that action if you don't master the AutoHotKey scripting language.

If the AutoHotkey script crashes, AHF may crash too.

This action has low performance, since a new AutoHotkey thread must be started.

Settings

AutoHotKey script

Script that will be executed.

Variables that should be exported prior to execution

Specify a list of variables that should be exported to the AutoHotKey script prior to execution.
The list can be separated either by a space, linefeed, comma or tab.
The variables will be available in global scope.

Variables that should be imported after execution

Specify a list of variables that should be imported from the AutoHotKey script after execution.
The list can be separated either by a space, linefeed, comma or tab.
The variables must be available in global scope and will be accessible in the flow with same.

Remarks


If the script reaches the bottom, it will end, even if it is persistent.
The script may be stopped with exitapp.
Do not use command onexit.
Do not use functions EnterCriticalSection() and LeaveCriticalSection() as well as other AHK-H specific functions.
Do not use variables which start with ahf_.
